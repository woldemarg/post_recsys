Designing custom recommender engine¶

Полученная матрица рейтингов описывает типичную задачу для РС:

  • некоторым пользователям можно рекомендовать к просмотру фильмы из нашей кинотеки, которые они еще не видели
  • индивидуальные списки рекомендаций нужно "ранжировать", т.е. представить по убыванию интереса для конкретного пользователя на основе рейтингов, которые спрогнозирует РС

Вся минимально необходиммая информация для решения описанных задачи есть в самой матрице рейтингов - это история оценивавания пользователями просмотренных других фильмом (memory-based РС).

В основе memory-based РС лежит предположение: оценку пользователя Y для неизвестного ему фильма Z можно спрогнозировать на основе оценок, присвоенных этому фильму Z другими пользователями из множества A:X, кино-предпочтения которых похожи на предпочтения пользователя Y.

В наиболее простом случае прогноз рейтинга для пары Y-Z можно расчитать как среднее взвешенное набора известных рейтингов фильма Z с учетом степени "похожести" пользователя Y на каждого из пользователей А:Х.

Степень "похожести" определяется расстоянием между векторами рейтингов этих пользователей (из матрицы рейтингов), приведенных к стандартной шкале путем вычитания среднего рейтинга каждого пользователя. В стандартизированном векторе рейтингов среднее значение будет равно нулю, что даст нам возможность заполнить все отсутствующие рейтинги (для неизвесных пользователю фильмов) тоже нулями.

Матрица косинусных расстояний между стандартизированными векторами рейтингов пользователей.

Все элементы главной диагонали матрицы равны нулю (расстояния от пользователя до себя самого).

Убираем главную диагональ и получаем для каждого пользователя вектор расстояний от него до всех других пользователей в системе. Внутри РС эти расстояния будут преобразрваны в "обратные" веса для расчета прогнозных средневзвешенных рейтингов.

rats_tiny_arr = rats_tiny_mtx.to_numpy()


def to_weights(dist_vec):
    # zero cosine distance means equality among users
    # replace with infinitely small value to assign bigger 'reverse' weight
    rv = [1 / np.finfo(np.float64).eps if x == 0 else 1 / x for x in dist_vec]
    sm = sum(rv)
    return np.array([x / sm for x in rv])


rates_est = np.array([])

for i in range(len(rats_tiny_arr)):

    rats_tiny_arr_no_cur_user = np.delete(rats_tiny_arr, i, axis=0)

    for cur_mov_rates_by_knn in rats_tiny_arr_no_cur_user.T:

        # skip cases with no actual ratings
        notna_idx = set(np.argwhere(~np.isnan(cur_mov_rates_by_knn)).flat)
        
        cur_user_dists_to_knn = users_dists_mtx_no_diagonal[i]

        # skip cases with negative cosine distances between users,
        # meaning opposite direction of their scales rates' vectors
        notneg_idx = set(np.argwhere(cur_user_dists_to_knn > 0).flat)

        knn_idx = list(notna_idx.intersection(notneg_idx))

        user_mov_rate_est = np.dot(
            cur_mov_rates_by_knn[knn_idx],
            to_weights(cur_user_dists_to_knn[knn_idx]))

        rates_est = np.append(rates_est, user_mov_rate_est)

rates_test_idx = np.argwhere(~np.isnan(rats_tiny_arr.flat))

custom_mape = mean_absolute_percentage_error(
    rats_tiny_arr.flat[rates_test_idx].ravel(),
    rates_est[rates_test_idx])

print('Biased MAPE for custom Recommender: {:.1%}'.format(custom_mape))

Production-ready recommender engines¶

Мы построили custom РС, которая успешно решает задачу прогнозирования рейтигов и позволяет на основе прогноза строить ранжированные списки рекомендаций.

Однако в том виде как есть такая РС имеет ряд недоработок:

  • не позволяет гибко переходить от user-based к item-based collaborative filtering
  • не позволяет тонко настроить количество "похожих" пользователей по рейтингах которых будет строиться прогноз
  • сложно перенастраивается для экпериментов с более продвинутыми алгоритмами прогнозирования
  • требует оптимизации для работы с большими объемами данных.

Всех этих недостатков лишены РС, которые можно развернуть на основе модуля surprise (https://surprise.readthedocs.io/en/stable/index.html)

Так, например, использование "коробочной" версии РС от surprise с чуть-более продвмнутым (но, в целом, близким к представленному выше) алгоритмом прогнозирования KNNWithMeans (https://surprise.readthedocs.io/en/stable/knn_inspired.html#surprise.prediction_algorithms.knns.KNNWithMeans) позволяет сразу снизить MAPE на той же крошечной матрице рейтингов c 15% до 11%:

Valuable recommendation: be careful with evaluation of recommenders¶

Для сравнения эффективности РС обычно используют не MAPE/MAE/RMSE, а метрики типа типа precision и recall для спискам объектов с прогнозными рейтингами, отфильтрованными по заданному threshold.

Так на примере нашей микро-матрицы рейтингов: если мы установим минимальный рейтинг фильма для попадания в список рекомендаций на уровне 4.0, то получим списки разной длины от разных РС для одних и тех же пользователей.

Мы рассмотрим как можно расчитать такие метрики, и как оптимизировать (и повысить эффективность) "коробочной" версии SVD-алгоритма (известного как Netflix-prize winner (https://pantelis.github.io/cs301/docs/common/lectures/recommenders/netflix/)) в следующей части.

Building memory efficient quasi-hybrid recommender engine: back to front (part 2)¶

В предыдущей части мы детально разобрали как работает memory-based РС изнутри.

Прогнозирование рейтингов так или иначе происходит на основе умножения вектора расстояний между пользователями и рейтинга выставленных ими рейтингов объектам внутри системы. Таким образом ключ к повшению точности прогноза, в большинстве случаев, заключается в повышении точности вектора расстояний (подборе дболее схожих пользователей) и/или в фильтрации более близких объектов (с учетом контента) улучшении качества прогноза нахождении соседей с максимально близкими поведенческими характестиками и/или в отфильтро лежит

class FrequencyEncoder(BaseEstimator, TransformerMixin):
    def __init__(self, cols):
        self.cols = cols
        self.ref_dic = None

    @staticmethod
    def _get_enc(ser):
        count = ser.value_counts()
        noise = abs(np.random.normal(5e-6, 1e-5, len(count)))
        return (count / count.sum() + noise).to_dict()

    def fit(self, X, y=None):
        self.ref_dic = {}
        for col in self.cols:
            col_enc = self._get_enc(X[col])
            self.ref_dic[col] = col_enc
        return self

    def transform(self, X, y=None):
        for col in self.cols:
            X[col] = X[col].apply(
                lambda x, col=col: self.ref_dic[col].get(x, 0))
        return X

# https://surprise.readthedocs.io/en/stable/FAQ.html#how-to-compute-precision-k-and-recall-k
def precision_recall_at_k(predictions, k=5, threshold=4):
    
    user_est_true = defaultdict(list)

    for uid, _, true_r, est, _ in predictions:
        user_est_true[uid].append((est, true_r))

    prec = dict()
    recs = dict()

    for uid, user_ratings in user_est_true.items():

        user_ratings.sort(key=lambda x: x[0], reverse=True)

        n_rel = sum((true_r >= threshold) for (_, true_r) in user_ratings)

        n_rec_k = sum((est >= threshold) for (est, _) in user_ratings[:k])

        n_rel_and_rec_k = sum(((true_r >= threshold) and (est >= threshold))
                              for (est, true_r) in user_ratings[:k])

        prec[uid] = n_rel_and_rec_k / n_rec_k if n_rec_k != 0 else 0

        recs[uid] = n_rel_and_rec_k / n_rel if n_rel != 0 else 0

    return prec, recs

def profile(fn):

    @wraps(fn)
    def inner(*args, **kwargs):

        fold = kwargs.pop('fold')
        case = kwargs.pop('case')

        tm = time.perf_counter()
        retval = fn(*args, **kwargs)
        elapsed = time.perf_counter() - tm

        mem, retval = memory_usage(
            (fn, args, kwargs), retval=True, timeout=200, interval=1e-7)

        res_metrics.append(Res(fold, case, elapsed, max(mem) - min(mem)))
        return retval

    return inner
